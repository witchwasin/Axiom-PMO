# Win Theme Guide — แนวทางเขียน Win Theme ใน Proposal

## Win Theme คืออะไร

Win Theme คือ **ข้อความ 1-2 ประโยคที่เชื่อม pain point ของลูกค้ากับ capability ของเรา** อย่างเฉพาะเจาะจง เป็น "กระดูกสันหลัง" ของ Proposal ทั้งฉบับ

Win Theme ไม่ใช่ slogan หรือคำขวัญ แต่เป็น argument ที่สอดแทรกอยู่ในทุก section

---

## โครงสร้างของ Win Theme ที่ดี

```
[Buyer's specific challenge] + [Our concrete capability] = [Measurable outcome]
```

### ตัวอย่าง: อ่อน vs แข็ง

| อ่อน (ห้ามใช้) | แข็ง (ใช้แบบนี้) |
|----------------|-----------------|
| "เรามีประสบการณ์ด้าน AI" | "ระบบ Visual Search ค้นหาอะไหล่จากภาพถ่ายได้ภายใน 2 วินาที ลดเวลาที่เซลส์ใช้ค้นจาก 5-10 นาทีเหลือแทบจะทันที" |
| "ระบบมีความปลอดภัยสูง" | "สถาปัตยกรรม Hybrid Intelligence ประมวลผลข้อมูลสินค้า ราคา และฐานลูกค้าภายใน Local Server ของบริษัท 100% — ไม่มีข้อมูลละเอียดอ่อนถูกส่งออกนอก" |
| "เรามีทีมงานคุณภาพ" | "ทีมเราเคย POC ผ่านกับภาพสินค้าจริง 1,000 รายการของลูกค้า ได้ผลลัพธ์ที่ลูกค้าพอใจภายใน 2 สัปดาห์" |

## Win Theme Matrix Template

สร้าง 3-5 Win Themes ต่อ Proposal:

```markdown
## Win Theme 1: [ข้อความ 1-2 ประโยค]
- **Pain Point ของลูกค้า:** [จาก MOM/REQ]
- **Capability ของเรา:** [เทคนิค/วิธีการ/ทีม]
- **หลักฐาน:** [metric, case study, POC result]
- **ปรากฏใน Section:** Executive Summary, Scope 3.1, Timeline Phase 1
```

---

## วิธีหา Win Theme จาก MOM/REQ

1. **อ่าน MOM ทุกฉบับ** — จด pain point ทุกข้อที่ลูกค้าพูดถึง
2. **จัดลำดับ** — ข้อไหนลูกค้าพูดซ้ำบ่อย / เน้นมาก = สำคัญกว่า
3. **จับคู่** — แต่ละ pain point จับคู่กับ capability ที่เราทำได้จริง
4. **เพิ่มหลักฐาน** — ใส่ metric หรือ fact ที่พิสูจน์ได้
5. **ทดสอบ** — ถ้าเปลี่ยนชื่อบริษัทแล้วข้อความยังใช้ได้ = อ่อนเกินไป ต้องเฉพาะเจาะจงกว่านี้

---

## กฎสำคัญ

- **ห้ามโจมตีคู่แข่งตรงๆ** — สร้าง contrast ผ่าน capability ของเรา ไม่ใช่โดยด้อยค่าคนอื่น
- **ห้ามใช้คำว่า "ครบวงจร" "ทันสมัย" "ชั้นนำ"** — แทนด้วย fact เฉพาะเจาะจง
- **Win Theme ต้องปรากฏอย่างน้อยใน 3 sections** — ถ้าอยู่แค่ที่เดียว จะไม่มีพลัง
