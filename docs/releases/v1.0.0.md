# Axiom-PMO 1.0.0 — Release Notes

**Axiom-PMO — The Anti-Hallucination Framework for AI Agents.** First public
release.

## What Axiom-PMO is

A deterministic governance control plane for AI-assisted software delivery. It
keeps AI coding agents inside verified requirements, approved scope, traceable
evidence, and human-controlled release gates — and it validates all of that with
a PowerShell engine that exits non-zero when a rule is broken.

## Why it exists

AI coding frameworks make agents build faster. They do not, on their own, stop an
agent from inventing requirements, expanding scope, claiming unverified evidence,
or shipping without permission. Axiom-PMO turns those boundaries into
machine-verifiable contracts. See
[the case study](../../case-studies/unauthorized-git-mutation.md) for the
incident that shaped the git-authority controls.

## Major capabilities

- **Anti-hallucination controls** — source references and evidence statuses on
  every requirement, decision, test, and release claim.
- **Risk-adaptive modes** — Lite / Standard / Strict, with effective mode that
  can be escalated but not silently downgraded.
- **Source-of-truth protection** — user-owned source is read-only to the agent.
- **Full-chain traceability** — source → requirement → design → delivery → test
  → evidence → release (RTM in Strict).
- **Human authority boundaries** — no self-approval; no commit/push/tag/deploy
  without explicit human confirmation.
- **Deterministic validation** — a doctor, a positive/negative fixture matrix,
  byte-for-byte golden masters, config-mutation tests, and generator-to-release
  end-to-end tests, run in CI.
- **Interoperability positioning** — designed to operate alongside AI execution
  frameworks as a control plane, with a documented Level 0–4 model.

## Quick start

```powershell
powershell -ExecutionPolicy Bypass -File scripts/new-project.ps1 -ProjectCode P02-DEMO -Mode Standard
powershell -ExecutionPolicy Bypass -File scripts/validate-project.ps1 -ProjectPath projects/P02-DEMO -Mode Standard -Gate Release -FailOnWarning
```

See [the first-project tutorial](../tutorials/first-project.md).

## Supported platforms

- Windows PowerShell 5.1 (used by CI).
- PowerShell 7 (`pwsh`).
- Linux/macOS via `pwsh` — **experimental** (wrappers run; the suite is not yet
  verified there).

## Compatibility model

Axiom-PMO is framework-agnostic. Interoperability **Levels 0–1** are available
today by convention; **Levels 2–3** ship as experimental, framework-neutral
schemas under `integrations/`; **Level 4** (automated bridge) is roadmap. See the
[interoperability overview](../integrations/overview.md).

## Upgrade notes

Coming from the previous private layout? The `pmo-` identifiers and validator
behavior are unchanged — see
[the migration note](../migration/from-pmo-template-personal.md).

## Known limitations

- Cross-platform execution via `pwsh` is experimental and not yet verified by the
  test suite.
- Interoperability Levels 3–4 are documented and specified as schemas but not
  automated.
- The framework performs a sensitive-file pre-check, not a full secret scan.

## Security expectations

Sensitive source stays local and triggers Strict mode. Report vulnerabilities
privately per [`SECURITY.md`](../../SECURITY.md).
