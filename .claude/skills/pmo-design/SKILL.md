# pmo-design

Use this skill for flow, UX, wireframe, and design-ready acceptance criteria.

Required inputs:

- `PROJECT.md`
- Relevant `DESIGN/` files
- Source only when design facts are missing or conflicting

Required outputs:

- Flow or design artifact references
- Acceptance criteria linked to requirement IDs
- Assumptions and open questions
- Design gate readiness notes

Do not create heavy diagrams for Lite work unless the change needs them.
