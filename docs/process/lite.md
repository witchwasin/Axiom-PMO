# Lite Process

Use Lite for bug fixes, small feature changes, and clarification tasks where a full design package would slow the team down.

## Flow

```text
Requirement -> Acceptance Criteria -> Develop -> Test -> Done
```

## Required Artifacts

- `PROJECT.md` section update or reference
- One delivery item in `DELIVERY.md` or one GitHub Issue
- Acceptance criteria
- Test note

## Rules

- Use `PROJECT.md` as the working context summary.
- Reopen source only when the summary is missing, disputed, or ambiguous.
- Do not create Use Case, full System Flow, or Wireframe unless the feature needs it.
- Log only scope changes, decisions, and meaningful risks.

## Exit Criteria

- Requirement has `source_ref`.
- Acceptance criteria are testable.
- No open blocker exists for the item.
- Human owner accepts the result.
